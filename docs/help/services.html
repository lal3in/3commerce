Platform services

The six DB-owning services added in ADR-0030 — Marketing, Pricing, Audit, Workflow, Entitlement, Usage — what each owns, every endpoint, and the use cases per option. All sit behind the gateway with internal-claims auth.

Six DB-owning services added once CI was restored (ADR-0030). Each owns a Postgres database + named schema, sits behind the gateway with internal-claims auth, and uses the MassTransit EF outbox. Two were extracted out of Fulfillment (Entitlement, Usage); two are event projections (Audit, Workflow); two are standalone domains (Marketing, Pricing).

Marketing (/api/marketing · :5108)

Campaigns and short links (mt5_1/5_3).

MethodPathAuthPurpose
GET/admin/campaignsadminList campaigns
POST/admin/campaignsadminCreate (cid, name, optional window)
POST/admin/campaigns/{id}/activate|pause|endadminLifecycle transitions
GET/admin/short-linksadminList short links
POST/admin/short-linksadminCreate; destination host must be a registered storefront host

Use cases

  • Launch a campaign with a tracking cid and an optional start/end window.
  • Pause or end a running campaign without deleting it.
  • Mint a short link for a campaign — rejected if the destination is not a known storefront host (anti open-redirect).

Pricing (/api/pricing · :5109)

The dedicated home for prices + graduated tiers (mt7_1). The Catalog Offer keeps a flat price (ADR-0028); this is what recurring/usage charging bills against.

MethodPathAuthPurpose
GET/admin/pricesadminList prices with tiers
POST/admin/pricesadminCreate: pricing_model + billing_period + amount + tiers
GET/admin/prices/{id}/quote?quantity=adminQuote a charge for a quantity

Use cases

  • Define a flat one-time price.
  • Define a subscription price with a billing period.
  • Define graduated tiers (first tier must start at qty 1) and quote the per-block charge for any quantity.

Audit (/api/audit · :5110)

Central searchable projection of every service’s local audit entry (mt6_1). Read-only; the owning service’s hash-chained log stays authoritative. Consumes AuditEntryRecorded.

MethodPathAuthPurpose
GET/admin/audit?tenantId=&resourceType=&resourceId=&action=&outcome=adminCross-service audit search (latest 200)

Use cases

  • Investigate who changed a supplier record across services.
  • Filter to denied/sensitive actions for a compliance review.
  • Trace a resource’s history by id.

Workflow (/api/workflow · :5111)

Central scheduled-job run history (mt6_3). Read-only projection of JobRunRecorded.

MethodPathAuthPurpose
GET/admin/workflow/runs?job=adminRecent runs (status, timing, error)

Use cases

  • Confirm the nightly Payments journal job ran and succeeded.
  • Find the error message from a failed scheduled run.
  • Filter run history to one job by name.

Entitlement (/api/entitlement · :5112)

Digital/service-line access (mt7_2/7_6), extracted from Fulfillment. Its EntitlementIssuingConsumer subscribes to OrderConfirmed and issues access for digital lines.

MethodPathAuthPurpose
GET/admin/entitlements?tenantId=&orderId=&email=adminList entitlements
GET/me/entitlementscustomerThe signed-in customer’s own access

Use cases

  • A confirmed order with a digital download issues an entitlement (no shipment).
  • A customer views their own access on the storefront “my access” page.
  • An operator checks what access an order granted.

Usage (/api/usage · :5113)

Metered usage + overage billing (mt7_4/7_5), extracted from Fulfillment. Publishes UsageOverageCharge → Payments.

MethodPathAuthPurpose
POST/admin/usage/provisionadminSet a meter’s included allowance + overage price
POST/admin/usage/recordadminRecord usage (idempotent by referenceId); gated when overage disallowed
POST/admin/usage/balances/{id}/bill-overageadminCharge unbilled overage (once)
GET/admin/usage/balances?email=adminList balances
GET/me/usagecustomerThe customer’s own balances

Use cases

  • Provision a plan with 1000 included calls and a per-unit overage price.
  • Record metered usage idempotently; reject usage when the allowance is exhausted and overage is off.
  • Bill the unbilled overage once at period end (re-billing is a no-op).
